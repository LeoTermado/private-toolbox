Download, install, and set up Ghostscript for the PDF Compressor.

Short answer

The PDF Compressor works best with Ghostscript.

Ghostscript gives the toolbox access to strong PDF compression. It can reduce file size much better than the basic fallback mode, especially for PDFs with images, scanned pages, exported slides, or large embedded media.

Without Ghostscript, the compressor still works, but it uses basic fallback compression. That fallback is weaker and may only reduce the file size a little. In some cases, the output file can even become slightly larger.

What Ghostscript is

Ghostscript is an external system program for working with PDF and PostScript files.

It is not a normal Python package. It should not be installed with pip, and it should not be added to requirements.txt.

This toolbox uses Ghostscript from the command line when it is available on your system.

On Windows, the important command-line executable is:

gswin64c.exe

The PDF Compressor automatically checks whether Ghostscript is available when you open the compressor page.

Why the PDF Compressor needs Ghostscript

PDF files can contain many different kinds of content:

  • normal text
  • vector graphics
  • embedded images
  • scanned pages
  • exported PowerPoint slides
  • metadata
  • fonts
  • already compressed objects

The basic Python fallback can only optimize some internal PDF streams.

Ghostscript can do stronger PDF optimization. It can rewrite the PDF and apply compression profiles that reduce file size in a controlled way. This is especially useful for PDFs that contain images or scanned pages.

That is why the PDF Compressor’s three compression levels work best with Ghostscript:

  • Low compression / best quality
  • Medium compression / balanced
  • High compression / smallest file

What happens without Ghostscript

If Ghostscript is not installed, the toolbox falls back to basic PyPDF2 compression.

This fallback is useful because the compressor can still run without extra system software, but it has limits.

The fallback usually cannot:

  • strongly downsample images
  • optimize scanned PDFs well
  • shrink exported slide decks as much as Ghostscript
  • guarantee that the output file becomes smaller

So if Ghostscript is missing, the compressor may show a warning and the result may be weaker.

Which version to install

Install the official Windows 64-bit AGPL version from the Ghostscript website.

For this project, we tested:

  • Ghostscript 10.07.1 for Windows (64 bit)
  • Ghostscript AGPL Release
  • Tested on: 04.06.2026

At the time of testing, Ghostscript 10.07.1 was the latest official release. The official Ghostscript release page listed Ghostscript 10.07.1 as released on 2026-05-19.

If a newer 10.x version is available on the official Ghostscript page, it should usually also work. The toolbox looks for the normal Ghostscript command-line executable and does not depend on one exact version number.

Choose:

  • Ghostscript 10.x.x for Windows (64 bit)
  • Ghostscript AGPL Release

Do not choose the commercial license version unless you specifically need a commercial Ghostscript license.

How to download Ghostscript

Open the official Ghostscript download page:

https://ghostscript.com/releases/gsdnld.html

On Windows, choose the Windows 64-bit AGPL download.

You should select:

  • Ghostscript 10.x.x for Windows (64 bit)
  • Ghostscript AGPL Release

Do not download from random third-party sites. Use the official Ghostscript website.

How to install Ghostscript on Windows

  1. Download the Windows 64-bit AGPL installer from:

    https://ghostscript.com/releases/gsdnld.html

  2. Run the downloaded installer.
  3. Keep the default installation path unless you have a reason to change it.

    It usually installs into a folder like:

    C:\Program Files\gs\gs10.07.1\
  4. During installation, leave optional font/support components enabled if they are already selected.

    Extra font support is not required for normal PDF compression, but it usually does not hurt.

  5. Finish the installation.

How to add Ghostscript to PATH manually

The PDF Compressor needs Windows to find this command:

gswin64c.exe

Usually, Ghostscript’s installer adds it to PATH automatically.

If it does not, add the Ghostscript bin folder manually.

The folder usually looks like this:

C:\Program Files\gs\gs10.07.1\bin

Replace gs10.07.1 with the version you installed.

Steps:

  1. Press the Windows key.
  2. Search for: Environment Variables
  3. Open: Edit the system environment variables
  4. Click: Environment Variables
  5. Select the Path variable.

    You can use either your user Path or the system Path.

  6. Click: Edit
  7. Click: New
  8. Add the Ghostscript bin folder, for example: 
    C:\Program Files\gs\gs10.07.1\bin
  9. Click OK until all windows are closed.
  10. Restart VS Code and all open terminals.

How to test the installation

Open a new PowerShell window.

First run:

where.exe gswin64c

Expected result:

C:\Program Files\gs\gs10.07.1\bin\gswin64c.exe

Then run:

gswin64c -version

Expected result:

GPL Ghostscript 10.07.1

The exact version number can be different if you installed a newer version.

If both commands work, Ghostscript is installed correctly and available on PATH.

What to do after installing Ghostscript

After installing Ghostscript, restart:

  • VS Code
  • your terminal
  • the Flask app

Then start the toolbox again:

python run.py

Open the PDF Compressor page again.

If Ghostscript was installed correctly, the status card should show that Ghostscript was detected and strong PDF compression is available.

If the compressor still does not detect Ghostscript

First, test Ghostscript in a new PowerShell window:

where.exe gswin64c

If this does not show a path, Ghostscript is either not installed correctly or its bin folder is not on PATH.

If the command works in PowerShell but the toolbox still does not detect Ghostscript:

  1. Stop the Flask app.
  2. Close VS Code.
  3. Open VS Code again.
  4. Open a new terminal.
  5. Start the app again: 
    python run.py

This is needed because the Flask app may have been started before Windows updated the PATH environment.

Why some PDFs do not shrink much

This is normal.

Compression results depend on the PDF content.

A PDF may already be optimized. It may contain mostly text or vector graphics. In those cases, there is not much left to compress.

For some PDFs, low and medium compression may look visually identical and still reduce the file size a lot. High compression may reduce the file a little more, but images can become visibly worse.

PDFs that usually shrink more:

  • scanned PDFs
  • image-heavy PDFs
  • PDFs exported from PowerPoint
  • PDFs with many photos
  • PDFs with large embedded media

PDFs that may not shrink much:

  • mostly text-based PDFs
  • already optimized PDFs
  • PDFs with simple vector graphics
  • very small PDFs

Recommended compression level

Use this rule:

Low compression    = best quality
Medium compression = recommended default
High compression   = smallest file, possible visible quality loss

For normal use, medium compression is the best default.

Use high compression only when file size matters more than image quality.

Important licensing note

This toolbox does not bundle Ghostscript.

Users install Ghostscript separately from the official Ghostscript website.

Ghostscript is available under the AGPL license or a commercial license. For normal local use of this toolbox, installing the AGPL release separately is the intended setup.

Do not copy the Ghostscript installer or Ghostscript program files into this GitHub project unless you understand the license obligations.
